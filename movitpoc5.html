Why this page cannot read “app data”
  • JavaScript runs in the web origin of this document (e.g. https://…), not inside Android app storage.
  • It has no access to SharedPreferences, SQLite, internal files, or other apps’ data unless the host WebView exposes a @JavascriptInterface that deliberately returns such data. In Moovit’s WebViewActivity, the bridge named mobile only exposes closeWebView()—not data APIs.
  • So this HTML cannot “pull” PII or card numbers from the app by itself. Any sensitive UI would only appear if you navigate the app to screens that show it (separate from this page reading storage).
About intent: buttons below
  • They only try to fire Android Intent URIs via window.location. Whether anything launches depends on WebView handling, exported components, and Android version—often non-exported activities will not start from outside.
  • PaymentAccountEditDetailsActivity expects a personalDetails Parcelable extra; starting it with a bare component Intent may crash or show an empty state—this is expected if required extras are missing.

How to load this page inside Moovit WebView (example)

Use the app’s deep link and URL-encode your page URL:

moovit://webview?url=ENCODED_HTTPS_URL_TO_THIS_FILE

Example (replace host and path):

moovit://webview?url=https%3A%2F%2Fexample.com%2Fpoc.html
